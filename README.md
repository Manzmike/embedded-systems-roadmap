# Embedded Systems Roadmap

A self-directed, mastery-based curriculum working toward firmware architecture 
at the hardware/software boundary — BSP, HAL, RTOS, bootloaders, and driver frameworks.

This repo documents my progress, solutions, and weekly build logs across a 
structured 4-phase curriculum targeting roles at SpaceX, Anduril, Apple, and NVIDIA.

---

## Structure

| Folder | Focus |
|---|---|
| `c-fundamentals` | C from first principles — memory, pointers, systems thinking |
| `cpp-systems` | C++ for embedded — RAII, templates, zero-cost abstractions |
| `os-internals` | OS concepts, Linux internals, device drivers |
| `embedded-firmware` | Bare-metal, RTOS, ARM Cortex-M, BSP development |
| `weekly-reports` | Weekly build logs — what was built, what broke, what was learned |

---

## Methodology

- Mastery before movement. Nothing advances until concept is proven through building.
- Every week: read, build, break, fix, document.
- Problems modeled after real interview problems from target companies.
- All hardware is owned and physical — no simulators.

STM32 Nucleo, ESP32-S3, MSP-EXP430, Arduino Mega 2560, Basys 3 FPGA,
